General remarks

When you search for a word or a phrase, the search engine yields 3-sentence excerpts from subtitles files containing you request.

The results are then ranked according to the proximity of the searched items: the closer these words are in excerpt, the higher this excerpt is in the results table.

Below each row of results you can find a link to the movie description card in the imdb.com and sometimes the title of the film in one or both languages.

Data for different language pairs is stored in different tables, so you may have different results for the same search string and different output languages.

To learn how the databases are analyzed and the search is implemented, view the technical note.

Search requests

Some examples
You can use the following commands in the search string:
  • take my breath away
    The result should contain all words from the request.
  • take|my|breath|away
    The result should include at least one of the words from the request.
  • "take my breath away"
    The result will contain the requested words one immediately followed by another.
  • "my breath"~N
    Looks for the words "my" and "breath" separated by no more than N other words.
  • my << breath
    The word "my" should precede the word "breath".
  • =take
    The result should contain the exact form "take", contrary to "takes", "took" etc. allowed by all the previous examples.
  • ="take a breath"
    All words should be present in a given form and follow one another.
You can also use parenthesis to group operators, e.g. ("bring into")|("take out") will result in sentences containing phrasal verbs "bring into" OR "take out".
POS tags

Most nouns, verbs, adjectives and adverbs are stored in the databases alongside with the corresponding part-of-speech (POS) tag. So, if you want to search for the word "light" as an adjective, you should perform the following query:

light_JJ

The following POS tags can be processed:
NN nouns
NP proper nouns
VB verbs
RB adverbs
JJ adjectives

For the monent, POS tags are not available for the Russian language.

Wildcards

Wildcards (*) are acceptable in search. The length of the rest of the word should not be smaller than 3 symbols: "work*" will result in words such as "working", "workable", "workload" etc., while "wi*" will be discarded as this request contains only two unmasked symbols.

Although it may be tempting to perform queries like *_NN or *_JJ to look for all nouns or all adjectives, such requests would result in a too long query time and, therefore, all requests like *_tag will be discarded.

Limitations

Unregistered users can make up to 50 queries per day, for registered users the number of queries is limited by 100. This is a preventive measure to protect the server from attacks and avoid overcharge.
